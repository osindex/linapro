## ADDED Requirements

### Requirement: 插件平台治理权限必须与租户插件自服务分离

系统 SHALL 区分插件平台治理权限和租户插件自服务权限。平台插件治理包括插件上传、同步、安装、卸载、启用、禁用、升级、安装模式切换、新租户自动启用策略和平台插件状态治理；这些操作 MUST 要求平台上下文。租户插件自服务只能通过租户插件接口作用于当前租户的 `tenant_scoped` 插件启用状态。

#### Scenario: 租户上下文不能执行平台插件治理动作

- **WHEN** 租户管理员具备历史或异常 `plugin:edit` 权限
- **AND** 调用插件安装、卸载、启用、禁用、同步、上传或升级等平台治理接口
- **THEN** 系统 MUST 返回结构化权限错误
- **AND** 不修改 `sys_plugin`、平台 `sys_plugin_state`、插件发布快照或插件治理资源

#### Scenario: 租户上下文使用租户插件自服务

- **WHEN** 租户管理员通过租户插件接口启用或禁用当前租户的 `tenant_scoped` 插件
- **THEN** 系统仅修改当前租户对应的插件启用状态
- **AND** 不修改平台安装模式或新租户自动启用策略
- **AND** 插件运行时缓存失效范围限定为受影响插件和租户

#### Scenario: 平台上下文执行插件治理动作

- **WHEN** 平台管理员处于平台上下文并具备对应插件治理权限
- **THEN** 系统允许执行平台插件治理动作
- **AND** 按既有插件生命周期、依赖、守卫、审计和缓存失效规则处理
