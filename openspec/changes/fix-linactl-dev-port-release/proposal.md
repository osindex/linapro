## 背景

当前`make dev`通过`linactl dev`重启本地后端和前端开发服务。命令会先停止`linactl`通过`temp/pids`记录的旧服务，再检查`BACKEND_PORT`和`FRONTEND_PORT`是否空闲。

实际使用中，旧服务被发出终止请求后，操作系统释放监听端口可能存在短暂延迟。当前实现停止后立即执行端口检查，可能在旧服务即将退出时误报`backend port 9120 and frontend port 5666 already in use`，让开发者误以为需要手动处理。

## 变更内容

- 让`linactl dev`在确实停止过由`linactl`管理的服务后，短暂等待这些已停止服务对应的端口释放。
- 保持对未知外部占用者的保护：如果端口占用方不是`linactl`通过`PID`文件管理的服务，命令仍然拒绝自动杀掉它，并提示停止占用者或换端口。
- 增加单元测试覆盖端口释放延迟场景和外部端口占用拒绝场景。

## 影响

本变更只影响本地开发工具`linactl dev`和`make dev`的重启编排，不改变生产后端行为、前端页面、公开`API`、数据库结构、缓存策略、数据权限边界或`i18n`资源。
