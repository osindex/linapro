## 目标

`make dev`应具备可预期的重启语义：当旧的后端或前端服务是当前仓库`linactl`此前启动并记录在`temp/pids`中的进程时，命令可以自动停止它们并等待端口释放；当端口由未知外部进程占用时，命令不得擅自终止该进程。

## 方案

1. 调整`devservice.StopService`返回值，使调用方可以知道是否成功向`PID`文件记录的服务发送了终止请求。
2. `linactl dev`复用停止流程后，如果至少停止过一个已管理服务，则在端口占用检查前对这些已停止服务对应的端口等待一个有限的释放窗口。
3. 等待逻辑复用现有跨平台`PortInUseFunc`探测能力，不引入`lsof`、`pgrep`、`kill`等平台专属命令。
4. 等待超时后仍沿用现有错误提示，要求开发者停止占用者或通过`BACKEND_PORT`、`FRONTEND_PORT`换端口。

## 边界

- 不自动终止未知端口占用者。
- 不修改`make stop`的用户可见语义；它仍只停止`linactl`通过`PID`文件管理的服务。
- 不新增长期维护的`shell`脚本或平台专属工具。

## 影响评估

- i18n：不新增或修改前端文案、菜单、路由、接口文档、插件清单或语言包资源。
- 缓存一致性：不涉及运行时缓存、分布式失效、共享修订号或跨实例状态。
- 数据权限：不新增或修改数据操作接口、查询、写入、下载、聚合或插件数据访问路径。
- 开发工具跨平台：本变更直接修改跨平台开发工具`linactl`，实现必须使用`Go`标准库和现有抽象，并通过`linactl`包测试验证。
